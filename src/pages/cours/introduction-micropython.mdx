## Qu'est-ce que MicroPython ?

MicroPython est une implémentation légère de Python 3, conçue pour tourner sur des microcontrôleurs avec peu de mémoire. Créé en 2013 par Damien George, maintenu par la communauté open source.

### Python vs MicroPython

## Les modules essentiels

### `machine` — contrôle du matériel

```python
from machine import Pin, PWM, ADC

# GPIO digital
led = Pin(2, Pin.OUT)
led.on()   # Allumer
led.off()  # Éteindre

# Bouton avec pull-up
btn = Pin(0, Pin.IN, Pin.PULL_UP)
print(btn.value())  # 0 = appuyé

# PWM (signal variable — pour servos, luminosité)
pwm = PWM(Pin(2))
pwm.freq(1000)   # 1 kHz
pwm.duty(512)    # 50% (plage 0-1023)
```

### `network` — WiFi

```python
import network

# Mode AP — l'ESP32 crée son propre réseau
ap = network.WLAN(network.AP_IF)
ap.active(True)
ap.config(essid='MonESP32', password='12345678')

# Mode STA — rejoindre un réseau existant
sta = network.WLAN(network.STA_IF)
sta.active(True)
sta.connect('MonWiFi', 'motdepasse')
while not sta.isconnected():
    pass
print('IP:', sta.ifconfig()[0])
```

### `time` — temporisation

```python
import time

time.sleep(1)        # Pause 1 seconde
time.sleep_ms(500)   # Pause 500 ms
t = time.ticks_ms()  # Compteur en ms
```

## Le REPL — ton meilleur outil

Le **REPL** est une console interactive directe sur l'ESP32. Connecte-toi avec Thonny et tape :

```python
>>> from machine import Pin
>>> led = Pin(2, Pin.OUT)
>>> led.on()    # La LED s'allume instantanément !
>>> led.off()
```

Pas besoin de compiler ni de rebooter — idéal pour tester et apprendre.

## Gérer les fichiers sur l'ESP32

L'ESP32 possède un système de fichiers (Flash 4 Mo). Deux fichiers sont exécutés **automatiquement** à chaque démarrage :

<div class="grid grid-cols-1 sm:grid-cols-2 gap-4 my-6">
  <div class="rounded-xl border border-primary/20 bg-primary/5 p-5">
    <h3 class="font-bold font-mono mb-2">boot.py</h3>
    <p class="text-sm text-base-content/60 leading-relaxed">S'exécute en premier. Réservé à la configuration : connexion WiFi, initialisation matérielle. Léger de préférence. Si absent, ignoré.</p>
  </div>
  <div class="rounded-xl border border-emerald-500/20 bg-emerald-500/5 p-5">
    <h3 class="font-bold font-mono mb-2">main.py</h3>
    <p class="text-sm text-base-content/60 leading-relaxed">Ton programme principal. S'exécute après boot.py. Si absent, l'ESP32 attend dans le REPL. C'est le fichier que tu crées pour tes projets.</p>
  </div>
</div>

**Ordre de démarrage :** reset → `boot.py` → `main.py` → REPL (si main.py absent ou terminé)

### Le module `os` — gérer les fichiers depuis le REPL

```python
import os

os.listdir()           # ['boot.py', 'main.py', 'lib']
os.mkdir('lib')        # créer un dossier
os.remove('old.py')    # supprimer un fichier
os.rename('a.py', 'b.py')  # renommer
os.stat('main.py')     # taille, dates
```

### Mémoire disponible

```python
import os, gc
gc.collect()
print("RAM libre :", gc.mem_free(), "octets")   # ex : 120000 octets

fs = os.statvfs('/')
print("Flash libre :", fs[3] * fs[0], "octets sur", fs[2] * fs[0], "octets")
```
