## Ce que tu vas créer

À la fin de cet atelier, ton ESP32 sera un **mini-serveur web** qui :
- Crée son propre réseau WiFi (point d'accès, pas besoin de box)
- Sert une page HTML avec des boutons ON/OFF
- Contrôle une LED physique connectée au GPIO2

Tu pourras accéder à la page depuis n'importe quel appareil (téléphone, tablette, ordi) connecté au WiFi de l'ESP32.

## Étapes de réalisation

### Étape 1 — Installer MicroPython sur l'ESP32

Si ce n'est pas déjà fait, commence par flasher MicroPython sur ta carte :

1. Télécharge le firmware depuis [micropython.org/download/esp32/](https://micropython.org/download/esp32/)
2. Installe `esptool` : `pip install esptool`
3. Flash le firmware :

<div class="rounded-xl overflow-hidden border border-base-200/60 bg-[#1a1b26] my-4">
  <div class="px-4 py-2 border-b border-white/8 text-white/50 text-xs font-mono">terminal</div>
  <pre class="p-4 text-xs font-mono text-white/70 leading-relaxed overflow-x-auto"><code>{`# Efface d'abord la mémoire flash
esptool.py --port /dev/ttyUSB0 erase_flash

# Puis flash MicroPython
esptool.py --port /dev/ttyUSB0 --baud 460800 write_flash -z 0x1000 esp32-firmware.bin`}</code></pre>
</div>

### Étape 2 — Câbler la LED

Suis le schéma ci-dessus. La LED intégrée de l'ESP32 est sur **GPIO2** — on peut aussi l'utiliser pour tester sans LED externe.

### Étape 3 — Copier le code

Utilise **Thonny** (recommandé pour les débutants) ou `mpremote` :

<div class="rounded-xl overflow-hidden border border-base-200/60 bg-[#1a1b26] my-4">
  <div class="px-4 py-2 border-b border-white/8 text-white/50 text-xs font-mono">terminal</div>
  <pre class="p-4 text-xs font-mono text-white/70 leading-relaxed overflow-x-auto"><code>{`# Avec mpremote
mpremote cp main.py :main.py
mpremote reset`}</code></pre>
</div>

### Étape 4 — Tester

1. L'ESP32 redémarre et crée un réseau WiFi `ESP32-Fablab`
2. Connecte ton téléphone au réseau (mot de passe : `micropython`)
3. Ouvre `http://192.168.4.1` dans le navigateur
4. Clique sur les boutons pour allumer/éteindre la LED !
